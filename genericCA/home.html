GenericCA

GenericCA is a 2D, three-state cellular automata simulator which allows the user to specify the rules under which cells survive and are born, as well as ageing rules. It also gives the possibility to add uncertainty in the rules, so that state transitions happen with certain probabilities at each iteration.
The aim is giving the user the possibility to explore emergence and the consequence of changing rules.

First, a formal definition of a Cellular Automaton is given. After that, elementary cellular automata are described in order to clarify the definition given and as an example of how complexity can emerge in even very s imple systems; then, the class of cellular automata representable in GenericCA is defined. The topic of emergence and self-organization within cellular automata is then explored and many applications of them are described, with videos showing the simulation running in GenericCA.

Usage

  • Press SPACE to toggle edit mode
  • Click to change the state of a cell while in edit mode
  • press T (in edit mode) to change the type of cell inserted
  • Press C to clear the board
  • Press R to randomly initialise the board
  • Press +/- to speed up/down execution
  • Press ENTER to go to configuration mode

States: Red represents state B, Green represents state C.
Configuration mode: Each field in the configuration screen represents the transition (born, survive) of a cell in a certain state depending on the number of 8-neighbours it has. The specified numbers (0 to 8) represent the number of neighbours that make that transition happen. Transitions from and to state B and C depend, respectively, on number of neighbours in state B and C; transition from state B to C depends on the number of neighbours in state C, transition from state C to B depends on the number of neighbours in state B. Ages parameter makes cells that should die go in an aged state instead, and in the following iterations evolve through the number of ages specified by that parameter. When the age reaches the maximum number of ages, the cell dies. When a cell is in an aged state it cannot change state until it dies.

Definition of a Cellular automaton

Formally, a cellular automaton can be represented with a quadruple (C, \(\Sigma\), N, \(\Phi\)), where C is the lattice, \(\Sigma\) is a finite set of discrete cell states, N is a neighbourhood function, \(\Phi\) is the local transition function.

The lattice C is a set of discrete cells, each characterized by its position and state, that can be one the values contained in \(\Sigma\).
N is a function $$N: C\times C\rightarrow \{True, False\}$$ that defines the neighbours of a cell (usually, it is assumed that N is a simmetric function, i.e \(N(c_1, c_2) = N(c_2, c_1) \forall c_1, c_2\in C\)); the neighborhood of a cell is therefore the set $$N_{c_i} = \{c\in C | N(c_i,c) = True\} $$ \(\Phi\) is a function of the state of the neighbourhood of a cell: $$\Phi: S^{|N|}\rightarrow S$$ Based on it, it is determined what is the state of the cell at the following discrete time step.
Each of the cells, taken singularly, can therefore be seen as a finite state machine, whose transition rules depend on the current state and on the neighbour's states.

An extension on the above is a stochastic cellular automaton. In a stochastic cellular automaton each of the possible transitions happen with a certain probability. Each cell can therefore be seen as a Markov chain.

The above definition allows to define an infinite number of different cellular automata, varying the sets C, \(\Sigma\), the neighbourhood and transition functions N and \(\Phi\) and the initial conditions.

To better understand what a cellular automaton is and why it is interesting in the study of artificial life, 1-D elementary cellular automata are considered first, and they are generalised up to the class of cellular automata that can be represented within the applet.

Elementary cellular automaton

The simplest type of cellular automaton is one dimensional, with two possible states for each cell, and defining the neighbourhood of each cell as the two cells adjacent to it. More formally, the neighbourhood function associates to each cell two neighbours and is symmetric, and the states are only \(\Sigma = \{Alive, Dead\}\).
Changing the state transition function \(\Phi\) it is possible to obtain different cellular automaton.

In this simple case, it is possible to enumerate all the possible choices for the function, which are 256. In fact, there are \(2^3=8\) possible states for the neighbourhood of a cell (domain of \( \Phi\)), which are:

  1. left neighbour dead, cell dead, right neighbour dead (000)
  2. left neighbour dead, cell dead, right neighbour alive (001)
  3. left neighbour dead, cell alive, right neighbour dead (010)
  4. left neighbour dead, cell alive, right neighbour alive (011)
  5. left neighbour alive, cell dead, right neighbour dead (100)
  6. left neighbour alive, cell dead, right neighbour alive (101)
  7. left neighbour alive, cell alive, right neighbour dead (110)
  8. left neighbour alive, cell alive, right neighbour alive (111)

For each of these possible neighborhood states of a cell it is possible to choose if the cell evolves to the state 0 or to the state 1; therefore, there are \(2^8\) possible choices for \(\Phi\), which originate this number of cellular automata. Each of the possible rules is usually represented with a number from 0 to 255; the transition function corresponding to rule \(i\) is obtained expressing \(i\) in binary base and intepreting its k-th digit as the transition (mathematically, the image) referring to the state of the neighbourhood which is represented by \(k\) in binary form (mathematically, domain; where a neighborhood in binary form is expressed as in the above list). Despite the simplicity of this type of cellular automata and of its rules, great complexity can, surprisingly, arise from them (to see the patterns that arise from each of the rules, see mathworld.wolfram.com page about elementary cellular automata; some examples are discussed below).

GenericCA cellular automata

The proposed applet enables to represent a particular class of stochastic cellular automata.
The lattice C is a 2D grid, in which cells have 8 neighbours each. The set of states \(S=\{0, B, C\}\) allows to represent in it three states. The representable transition functions are those in which the next state of a cell depends on the number of neighbours of each state it has, i.e. on the number of neighbours in state 0, B, C. In order not to make the choice of parameters too complex (which would have brought to lack on intuition on how to make properties emerge from local rules) some simplifications have been made. They are explained below.

The possible transitions are:

  • No birth: \(0\rightarrow 0\)
  • B is born: \(0\rightarrow B\)
  • C is born: \(0\rightarrow C\)
  • B dies: \(B\rightarrow 0\)
  • B survives: \(B\rightarrow B\)
  • B transforms to C: \(B\rightarrow C\)
  • C dies: \(C\rightarrow 0\)
  • C transforms to B: \(C\rightarrow B\)
  • C survives \(C\rightarrow C\)
Note that it is not necessary to define, for each state of a cell, the rules under which each of the three transitions takes place: if two are specified, the third one follows from the others, as the default one. For simplicity, the transformations \(0\rightarrow 0\), \(B \rightarrow 0\), \(C \rightarrow 0\) are, respectively, assumed as default for each initial state.
Therefore, for the remaining 6 transformations the number of B and C neighbours (number of neighbours in state 0 follows as \(|0| =9-(|B|+|C)\)) under which the transition takes place has to be specified. As a simplification, it has been assumed that:
  • transitions \( 0 \rightarrow B \) and \(0 \rightarrow C \) depend only on the number of neighbours in state, respectively, B and C, and analagously the transitions \(B\rightarrow B\) and \(C \rightarrow C\);
  • transition \(C \rightarrow B\) depends only on the number of neighbours in state B, whereas the transition \(B \rightarrow C \) depends on the number of neighbours in state C.
In case of ambiguity, in which two transitions could possibly take place, it has been decided that:
  • transition \(0\rightarrow B\) has precedence over transition \(0 \rightarrow C\)
  • transitions \(B \rightarrow C\) and \(C \rightarrow B\) have precedence over transitions \(B \rightarrow B \) and \(C\rightarrow C\);
  • transition \(C \rightarrow B \) has precedence over transition \(B \rightarrow C \).
Note that the precedence rules advantage state B over state C, giving therefore an interpretation of "predator" to B and "prey" to state C in a model in which the states represent species.

For each of these transitions is possible to specify a probability; this is the aspect that makes the cellular automaton stochastic.

In addition, an ages parameter can be specified. When \(ages \gt 1\), whenever a cell should die it ages before that, i.e. it goes to a different, "aged" state. All cells in the regular states (0, B, C) above defined have an age equal to 0; when they go to the aged state their age is incremented, thus producing a new state. Cells in aged states follow the following simple transition rule:

  • If \(age+1\lt ages\) then increment age;
  • else reset the cell (age = 0; state = 0)
Adding the aging rule, despite its semplicity, completely changes the cellular automata that result from a certain set of rules, as will be shown in the examples.

Applications of cellular automata

Ilachininski (2001) identifies 4 principal fields of application of Cellular automata, that motivate their study:
  1. As powerful computation engines
  2. As discrete dynamical system simulators
  3. As conceptual vehicles for studying pattern formation and complexity
  4. As original models of fundamental physics

Second and third application fields are most relevant to Artificial Life; in particular, application number 3 has to do with emergence, which is a particularly interesting topic in the study of living systems. C. Langton (1986) in Studying Artificial Life with Cellular Automata seeks in cellular automata a means to explain how a living organism, despite being made of inanimate parts (molecules) that interact with each other according to physical laws, appear to be much more than the sum of them; for this reason, he compares study cellular automata to biochemistry, calling it artificial biochemistry, in which cells play the role of artificial molecules.
The hypothesis of the possibility of replicating life in cellular automata arrives up to the point to consider that intelligence could emerge in them: since the brain is a parallel aggregate system, it can be hypothesised that its dynamics involve the interaction of virtual automata in some form.

Chopard and Droz, in Cellular Automata Modeling of Physical Systems, explore in detail how physical systems can be modelled using cellular automata, as an alternative to use of more complex mathematical tools, like nonlinear partial differential equations. Many possible models of physical systems are explored, such as annealing, sand pile falling, ant movement, road traffic, solid body motion, gas particles movement, diffusion. This gives a very good idea of how many diverse models can be designed by use of cellular automata.

Addition of a probabilistic element furtherly improves the modelling power of cellular automata, adding the possibility to consider uncertainty inside of them. This is also appealing from an Artificial Life point of view, because it allows to add unpredictability to the automaton.

Emergence and Cellular automata

Self Organization

Many different physical and biological systems exhibit self-organization. Self-organization can be defined as formation of patterns occuring through interaction internal to the system, without intervention of external directing influences.

Camazine et al., in Self-Organization in Biological Systems (2001) define it as a process in which pattern at the global level of a system emerges solely from numerous interactions among the lower-level components of the system. Moreover, the rules specifying interactions among the system's components are executed using only local information, without reference to the global pattern.
This is strictly connected to emergent behaviour, in which an high level organization emerges from simple rules. Fundamental prerequisite in this is that there must not be any externally imposed instructions which globally guide the behaviour of the components.

Self organization and emergent behaviour are of particular interest in study of artificial life. Many living systems exhibit self-organization and emergence properties at different levels; examples are ant colonies, whose global path-seeking emerges from simple rules based on local concentrations of pheromone, and flocking birds, that adjust their velocity depending on their local neighborhood and produce a global structure (as pointed out by Craig Reynolds' Boid model).

Maruyama (1963) shows how great complexity can arise from very simple rules, and how a slight variation on these rules can produce big changes in the overall result. He applies this concept to give an explanation of how living beings have high complexity despite the relatively small amount of information stored in genes: genes carry just a set of rules to generate the complexity, which arises from those. He uses a particular 2D cellular automaton to exemplify that, showing how changes in rules and initial conditions originate big variations in the final outcome.

Self-organization can also emerge in non-living systems, based on laws of physics or chemistry. Examples of this are ripples in the sand, waves in the sea, crystallization.
Many systems can, in general, be seen from a self-organization point of view. An interesting example is car traffic: a global pattern emerges from local interaction of the drivers, as described by Boris Kerner (2004) in his Three-phase traffic theory, in which he identifies three phases of traffic (Free flow, Synchronized flow, Wide moving jam); global phases emerge from the individuals, without central organization.

Notably, very simple interaction among the basic components of a system can give raise to surprisingly complex global patterns; therefore, it eliminates the drawbacks associated with the approaches to study of artificial life based on global coordination. Real living systems are usually too complex for their behaviour to be specificated with an explicit, exhaustive description; and even when some of those rules are found, they will probably lack of flexibility. Therefore, finding a small set of rules from which the complex, global behaviour emerges from local interaction is particularly appealing in the attempt to produce life-like behaviours.

Self-organization and emergence in cellular automata

John Conway's game of life is probably the most well-known example of how complexity can emerge from simple rules. It is a 2-D cellular automaton based on a very simple set of deterministic rules in a rectangular grid: cells can have two states, alive or dead; when a cell is dead, it comes to life when it has 3 alive neighbours; when a cell is alive, it survives if it has 2 or 3 neighbours; otherwise, it dies.
Despite its simplicity, this set of rules generates a system in which there is complexity of behavior and ordered structures, up to the point that a universal Turing machine can be built inside of it (Life universal computer).

A cellular automaton-based model of traffic has also been developed (Kerner, 2002) to show how the global behaviour in traffic is actually emergent from local components of the system; this provides another example of how emergency can be modelled via cellular automata.

Stephen Wolfram in A New Kind of Science (2002) argues that the results of computation could bring to an understanding of the physical world; cellular automata are used as examples of very simple computational models in which great complexity is achieved. Konrad Zuse (1969) in his book Calculating Space even proposed that the law of the universe could be discrete and computable, and in particular he focused on cellular automata as models that could possibly describe how computation would take place in such an universe.

Classes of cellular automata

Stephen Wolfram studied elementary automata in detail as simple programs, which represent basic models of computation. He made a classification of Cellular Automata which helps understand how life-like behaviour can emerge from them. He divided cellular automata in four classes:
  1. Uniform. After a certain number of iterations, all the cells are constant, for nearly all the possible initial patterns. Elementary cellular automaton rule 1 is an example of a constant pattern.
  2. Periodic. After a certain number of iterations, the initial state of the cells evolves into structures that oscillate periodically. Rule 190 is an example of an oscillating pattern:

  3. Chaotic. There is no apparent pattern. It is worth noting how even extremely simple rules can generate chaotic and seemingly random patterns. An example is elementary cellular automaton rule 30, chaotic to the point that it has been used as a random number generator:

  4. Complex. There are oscillating and uniform patterns, as well as a certain amount of randomness. The way in which structure appears, for example, can be apparently random. This produces a complex evolution of the patterns. Elementary cellular automaton rule 110 is an example of such a cellular automaton:
    5. Among all the elementary cellular automata, rule 110 is the only one that has been proven to be Turing complete (by Matthew Cook). This is a particularly important result, since it proves that from a very simple rule follow universal computation capabilities.
The edge of chaos

From an artificial life point of view class 4 cellular automata are the most interesting. In fact, it can be argued that a characteristic of living system is complexity. It has been suggested that life is placed somewhere between order and chaos; Christopher Langton called this border "edge of chaos", in which all the complex, interesting behaviour takes place.
To help predict what the behaviour of a cellular automata will be depending on its rule set, he defined a parameter \(\lambda\) given by the fraction of rules in which the new state of a cell is living. When \(\lambda\) is 0, no cell is born, so the configuration is extremely ordered; when it approaches 1, almost all the rules cause life, so the behaviour is chaotic. He observed that, increasing the value of \(\lambda\) from 0 to 1, initially the CA is highly ordered, then there is a transition from order to chaos, and finally a chaotic behaviour. It is near the transition that the most interesting CA are usually found: when there is order, the effect of perturbations die out; when there is chaos, perturbations propagate very rapidly. When there is complexity, perturbations evolve, with information of the perturbation propagating inside the system. As Andrew Ilachinski (2001) points out, at the edge of chaos information can both propagate over long distances without appreciable decay and store this information, without neither locking into an ordered structure nor dissolving into randomness, and this is what gives raise to complexity.

Models with cellular automata

Game of Life

As mentioned above, John Conway's Game of Life is one of the most well-known and well-studied cellular automata. It belongs to class 4 cellular automata in Wolfram's classification; it exhibits complex behaviour, while being described by very simple rules. There are many still lives (stable patterns), oscillators, chaotic patterns, gliders. The number of different possible patterns is surprisingly high; LifeWiki contains a collection of many of the studied ones. To have an idea of the variety of different patterns possible, it can be noted that the different patterns among only 24-cell strict (i.e. connected) still life ones are 4051711. Very complex patterns can emerge from it; just a few of the many possible examples of this are glider guns, 4-8-12 diamond and corderships.
For these reasons, Game of Life has often been taken as an example of how an apparent design could emerge without the presence of a designer, just by applying a set of simple, deterministic rules.
Game of Life can be reproduced, in its original version, within the applet as a particular case, by setting the rules: Born B:3, Survive B:23, Initial proportion C:0, ages:1, random birth probabilities to 0 and leaving born, survive, death probabilities to the default values of 1.

More variability over Game of Life could be obtained considering a noisy version of it, varying probabilities for state transitions. The idea of a stochastic Game of Life is appealing because many biological systems exhibit some form of indeterminism.

Monetti et al. (1997) studied systematically how variation in state-transition probabilities in a simple 1-D cellular automaton bring changes to the behaviour of the system, making the initial state end into extinction, overcrowding, chaotic life or frozen life.
In their model, each chell has four neighbours, and stays alive when it has four live neighbours. A cell remains alive when it has two live neighbours with probability \(p_r\), when it has one or three live neighbours with probability \(p_s\); the transition areas between these extreme conditions are the most interesting, as in those a complex behaviour might emerge:

Analysing in detail all the possible combinations of probabilities and their consequences in a complete, stochastic game of life is much more complex, since there are many more parameters. However, experimentation within the applet has suggested that a born/survive/death probability of 0.9999 and a random birth probability of 0.0001 originate a cellular automaton producing more varied patterns from an initially random state than deterministic game of life, and also prevent the cellular automaton to evolve to a still-life asymptotic status.

Predator-prey game of life model

Another variation is a two-species game of life, which can be represented with a 3-state cellular automaton. A species (e.g. state B) could be interpreted as "predator", while the other one (state C) could be interepreted as "prey". This brings to also define a transition rule C to B, that could happen with a certain probability (e.g. 0.2) for a number of neighbours in state B greater than 1. This transition could be interpreted as predation.
To model the fact that prey species does not need to attack other species to survive, a probability of random birth of 0.0001 could be given to it; on the other hand, a probability of survival of 0.9999 could be given to predator species (all other probabilities are left to deterministic values).

This confirms how the proposed model is a realistic representation of predation dynamics.

HighLife

A famous variation on Game of Life is HighLife. It has the same rule set as Game of Life with the addition of the rules that cells are born when they have 6 neighbours. This brings to a cellular automaton very similar to Game of Life, but in which there is a very simple replicator pattern, which leads to additional complexity with respect to normal Life rules.

Disease spread

Another interesting model that can be simulated using the applet is the spread of a disease.
In this case, states will be given by \(\Sigma = \{dead, infected, healthy\}\).
Dead cells become alive when they have (in this example; the dynamics of the population modelled can be varied changing these rules) 2, 3, or 4 alive neighbours, with probability \(p_{b,i}\) from infected state, and \(p_{b,h}\) from healthy state. Transition from alive to dead cells happens, irrespective of the number of neighbours, with a probability \(p_{d,h}\) (probability of death of healthy cells) from healthy cells, \(p_{d,i}\) (probability of death of infected cells) from infected cells. A healthy cell can become infected when it has at least an infected neighbour, with probability \(p_{hi}\) (probability of infection); an infected cell can go back to healthy state with probability \(p_{ih}\) (probability of healing), irrespective of the number of healthy neighbours.
Changing the values of the probabilities gives raise to different possible models of disease spread. For example, values of \(p_{b,i} = 0.0001\), \(p_{b,h} = 0.04\), \(p_{d,h}=0.02\), \(p_{d,i}=0.05\), \(p_{hi}=0.1\), \(p_{ih}=0\) models an incurable, rather contageous, moderately deadly disease. Forest fire model

Another model well represented by use of probabilistic cellular automata is forest fire. States in this case are \(\Sigma = \{empty, green tree, burning tree\}\); a green tree that has at least one burning tree as a neighbour becomes a burning tree, and a burning tree becomes an empty cell. Green trees grow in empty sites with probability \(p\), and trees burn with probability \(f\) (lightning probability). Forest fire model can therefore be simulated inside the applet.

Forest fire model is of particular interest because it exhibits self-organised criticality (in steady state, characteristic quantities of the system follow a power law); a mathematical model of forest fire is very complex, whereas a cellular automaton based simulation can be created with very simple rules.

Conclusions

A large class of cellular automata can be modelled with the proposed applet; by changing its parameters, an incredible diversity of behaviours can emerge. In this page a few interesting set of rules have been pointed out, which are only a small subset of all the possible models obtainable changing the rules. The aim of the applet is allowing the user to find even more interesting ones trying different rule sets and observing the behaviours that emerge from them.

Many more, even more complex, cellular automata could be conceived. For example, neighbourhoods of different shapes or non-square cells could be considered, more than two states could be allowed, the grid could be 3-dimensional, more general rules could be used. A lot can, however, be learnt about emergence and self-organization through the proposed application, of which these variations could be considered further extensions. The models shown above are just a small part of all the possible ones that could be conceived, and they have been selected to demonstrate the modelling power and flexibility of cellular automata.
